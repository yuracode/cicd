# 技術研究カリキュラム（React → テスト → CI/CD → デプロイ）

このリポジトリは、専門学校ICT学科「技術研究」授業（全30コマ）の授業計画を、1コマ1ファイルのMarkdownで管理するためのものです。

学習テーマは以下の流れで一気通貫に設計されています。

- Reactで画面を作る
- テストで品質を担保する
- GitHub Actionsで自動化する
- CI/CDパイプラインとして運用する
- デプロイして公開・発表する

---

## このカリキュラムの狙い

学生が「ローカルで動いた」で終わらず、

- 実装
- テスト
- 自動チェック
- 自動デプロイ
- 説明・発表

までを自分の手で通せる状態になることを目標にしています。

---

## 全体の流れ（30コマ）

### Phase 1（コマ1〜6）
React基盤づくり

- コンポーネント、props、stateの基礎を固める
- TODOアプリを実装する
- GitHub連携、ブランチ、PRの基本運用を体験する

### Phase 2（コマ7〜12）
テストの書き方

- テストの目的と種類を理解する
- Vitestで単体テストを実装する
- React Testing LibraryでUIテストを実装する
- モック、非同期テスト、カバレッジ計測を扱う

### Phase 3（コマ13〜18）
GitHub Actions入門

- YAMLとワークフローの基本を理解する
- Lint/TestをCIで自動実行する
- ブランチ保護、Secrets管理を導入する

### Phase 4（コマ19〜24）
CI/CDパイプライン構築

- デプロイ先（GitHub Pages / Vercel）を比較・選定する
- 自動デプロイを構築する
- パイプラインを一気通貫で完成させる
- デバッグ演習、チーム開発シミュレーションで運用力を上げる

### Phase 5（コマ25〜30）
仕上げ・発表

- 個人制作の企画・設計・実装を進める
- テスト、CI/CDを自分のプロジェクトに適用する
- READMEと発表資料を整え、最終発表で成果を説明する

---

## 学習としての接続イメージ

この授業の設計は、次のように段階的に接続されています。

1. UIを作れるようになる（Phase 1）
2. 壊れていないことをコードで証明できる（Phase 2）
3. その証明を自動実行できる（Phase 3）
4. 自動で公開までつなげる（Phase 4）
5. 自分の成果として説明できる（Phase 5）

単発スキルではなく、開発の実務フローを通しで体験する構成です。

## 対象学生の前提

- React: ほぼ初学者
- GitHub Actions: 初学者
- JavaScript / Node.js: 既習
- 環境: Windows + WSL2 Ubuntu

この前提に合わせ、実装と運用を段階的に積み上げる難易度になっています。
