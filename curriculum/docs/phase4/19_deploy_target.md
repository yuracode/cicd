##  本題

### 1. 静的ホスティングとは

**静的ファイル**（HTML / CSS / JS / 画像）だけを配信するサービス。

Reactアプリを `npm run build` すると `dist/` に静的ファイルが出力される。これをWebサーバに置くだけで配信できる。

**静的ホスティングの特徴：**

- **速い**（CDNで世界中にキャッシュされる）
- **安い**（無料枠が広い／リクエスト課金）
- **設定が簡単**
- **動的な処理ができない**（APIは別途用意）

**Reactアプリはほぼ静的サイトとしてデプロイできる**（APIは別サービス、例えば Vercel Functions / Firebase / 外部APIなど）。

### 2. 動的ホスティングとは

サーバ側でプログラムを実行できるタイプ。

- Render, Heroku, AWS EC2, Google Cloud Run など
- バックエンドAPI／サーバサイドレンダリングを伴う場合に必要

**この授業では扱わない**（時間の関係と、Reactの基礎学習に静的で十分）。

### 3. 主要デプロイ先の比較

| サービス | 向き | 特徴 | 料金 |
|---------|------|------|------|
| **GitHub Pages** | 静的 | GitHubに統合・シンプル | 無料（publicリポジトリ） |
| **Vercel** | 静的＋API | React/Next.js作者の会社・PRプレビュー強い | 個人無料 |
| **Netlify** | 静的＋API | Vercelの老舗版・フォーム機能など | 個人無料 |
| **Cloudflare Pages** | 静的＋API | CDN最強・同時デプロイ数無制限 | 個人無料 |
| **Firebase Hosting** | 静的 | Google製・DB/認証と統合しやすい | 無料枠あり |
| **AWS S3 + CloudFront** | 静的 | 企業定番・設定難しめ | 従量課金 |

**主要サービスの特徴詳細：**

**GitHub Pages**
-  GitHub一択で完結
-  設定がシンプル
-  URLは `ユーザー名.github.io/リポジトリ名`（Vite設定で base path 調整必要）
-  動的APIはなし

**Vercel**
-  **PRごとにプレビューURL自動発行** が強力
-  独自ドメインも簡単
-  Serverless Functions 使える
-  無料枠を超えると有料

**Netlify**
-  Vercelとほぼ同等
-  Build Plugins が豊富
-  Vercelと比べて最近やや影薄め

**Cloudflare Pages**
-  CDN性能・無料枠が圧倒的
-  一緒に Workers（エッジ関数）使える
-  設定画面がやや複雑

### 4. なぜ GitHub Pages → Vercel の順で学ぶか

**GitHub Pages を先にやる理由：**

- GitHub Actions との連携を **手作業で** 書くので、CI/CDの全体像が見える
- 認証なし・環境ごとで完結 → 最初にやるのに最適

**次に Vercel をやる理由：**

- **PRプレビュー** の価値を体感してもらう（実務では Vercel 採用が主流）
- 外部サービス連携、Secretsの実用例

**両方やることで：**

- 「自分でワークフロー書いてデプロイ」 vs 「SaaSに任せる」 の違いが分かる
- 現場では規模や要件で使い分ける判断ができるようになる

### 応用課題（推奨）

4. **静的ホスティングでは実現できないこと** を3つ考えて書く。例：
   - DBにユーザー情報を保存してログイン認証
   - サーバー側でAPIキーを使って外部サービスを呼ぶ
   - 巨大ファイルを受け取ってサーバー側で変換

→ できないこと／できることの線引きを自分の頭で引けると、将来「サーバーサイドが必要な瞬間」が分かる。

5. **Cloudflare Pages** または **Netlify** のどちらか1つの無料プランを調べて、既存の比較表（GitHub Pages / Vercel）に追加する。同じ軸（無料枠・機能・学習コスト等）で比較

### チャレンジ課題（挑戦）

6. **エッジ（Edge）** という概念を調べ、以下を自分の言葉でまとめる：
   - 従来のサーバー配信と「エッジ配信」の違い
   - Vercel / Cloudflare が「グローバルCDNのエッジで実行」と言う時、具体的に何が起きているか

→ 次のステージ（サーバーサイド・API開発）への橋渡し。

7. **「Vercel が突然サービス終了した時」** を想定し、自分のアプリをどう退避させるか手順を紙上で書く（リポジトリは残る／別ホスティングへの移行手順／DNS 変更）。**ベンダーロックイン** の観点を持つ練習
