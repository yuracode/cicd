### 2. 相互評価シート（学生用）

```markdown
## 発表者：○○さん

- ❓ 質問（あれば）：
```

全員分これを書きながら聞く。

### 3. 質疑応答のコツ

**質問する側：**

- 「Xの部分、どうしてそう実装したんですか？」（意図を聞く）
- 「Xで詰まったと言っていましたが、どう解決しましたか？」（経験を引き出す）
- ❌「ここはもっとXしたほうが良いのでは？」（発表の場では不要）

**答える側：**

- **分からない時は「分かりません」で良い**。次までに調べる
- 質問の意図を聞き返してOK
- 時間があれば少し掘り下げて話す

### 5. 次に学ぶこと（15分）

このコースの続きとして、興味に応じて挑戦してほしい領域：

**① フロントエンドを深める**

- **TypeScript**：大きなアプリでは必須レベル
- **Next.js**：Reactベース。SSR・API・SEO・ルーティングが揃う
- **状態管理**：Zustand / Jotai / Redux Toolkit
- **CSSフレームワーク**：Tailwind CSS / CSS Modules

**② バックエンドに手を伸ばす**

- **Node.js + Express / Hono**：自分でAPIを書く
- **Supabase / Firebase**：BaaS でDBと認証が手軽に
- **Prisma**：型安全なDBアクセス

**③ CI/CD・DevOps を深める**

- **E2Eテスト**：Playwright でブラウザを自動操作
- **Docker**：環境差をコンテナで封じる
- **Terraform / IaC**：インフラもコード化
- **モニタリング**：Sentry でエラー監視、GA4 で利用状況

**④ 学習の習慣化**

- 毎月1つ小さなアプリを公開する（発表アプリをひたすら改善でもOK）
- 仕事でコードを書く前に、自分のプロジェクトで試す
- 知らないキーワードが出たら深追いする
